FIX 5 — FAQ SCHEMA ON CONDITION PAGES (HIGH SEO VALUE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PROBLEM: No FAQ structured data exists on condition pages or the homepage. FAQ schema 
can generate rich results (expandable Q&A in Google SERPs) which dramatically improve 
click-through rates for health queries.

FIX:
Create a reusable FAQSchema component:

  interface FAQItem {
    question: string;
    answer: string;
  }

  export function FAQSchema({ faqs }: { faqs: FAQItem[] }) {
    const schema = {
      "@context": "https://schema.org",
      "@type": "FAQPage",
      "mainEntity": faqs.map(faq => ({
        "@type": "Question",
        "name": faq.question,
        "acceptedAnswer": {
          "@type": "Answer",
          "text": faq.answer
        }
      }))
    };
    return (
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(schema) }}
      />
    );
  }

Add this component to each condition page with 4–6 real FAQs. Example for PCOS:
  Q: "What foods should I avoid with PCOS?"
  Q: "Is rice bad for PCOS?"
  Q: "What is a good Indian breakfast for PCOS?"
  Q: "How many calories should a PCOS meal plan have?"
  Q: "Can I eat roti with PCOS?"

Also add a visible FAQ section to the page UI (accordion component) that matches — 
the schema and the visible content must be consistent. Hidden FAQ content will be 
ignored or penalised by Google.

Create a reusable Breadcrumb schema component:

  export function BreadcrumbSchema({ items }: { items: {name:string, url:string}[] }) {
    const schema = {
      "@context": "https://schema.org",
      "@type": "BreadcrumbList",
      "itemListElement": items.map((item, i) => ({
        "@type": "ListItem",
        "position": i + 1,
        "name": item.name,
        "item": item.url
      }))
    };
    return <script type="application/ld+json" 
      dangerouslySetInnerHTML={{ __html: JSON.stringify(schema) }} />;
  }
