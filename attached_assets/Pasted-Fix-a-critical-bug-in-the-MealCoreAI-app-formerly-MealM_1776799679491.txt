Fix a critical bug in the MealCoreAI app (formerly 
MealMate). The repo is a pnpm monorepo:
- artifacts/mealmate/ — React 19 + Vite frontend
- artifacts/api-server/ — Express 5 TypeScript backend
- lib/db/ — Drizzle ORM (profilesTable, dishesTable, 
  mealPlansTable, dishPreferencesTable)

THE BUG:
When a user changes their region in the profile page 
(e.g. from Northeast to North India), the profile 
updates correctly in the database but the dashboard 
still shows dishes from the OLD region because the 
existing active meal plan is not refreshed.

The REGION_MAP in meal-plans.ts works correctly 
during plan generation. The problem is that after a 
profile region change, no new plan is generated. 
The old plan with old regional dishes remains active.

Same bug applies when user changes:
- Region (Northeast → North India → South India etc.)
- Primary health track (PCOS → Diabetes etc.)
- Diet type (Veg → Non-Veg etc.)

══════════════════════════════════════
FIX 1 — DATABASE: Add regionUpdatedAt field
══════════════════════════════════════

In lib/db/src/schema/profiles.ts add a new column:
planInvalidatedAt: timestamp('plan_invalidated_at')

This timestamp records the last time any profile 
change was made that should trigger a new meal plan. 
Defaults to null.

Run the Drizzle migration after adding this column:
pnpm --filter @mealcoreai/db drizzle-kit generate
pnpm --filter @mealcoreai/db drizzle-kit migrate

Also update filterDishesByRegion() to handle the 
case where REGION_MAP returns undefined for an 
unknown region value — fall back to returning all 
dishes rather than returning empty array:

function filterDishesByRegion(
  dishes: Dish[], 
  region: string | null | undefined
): Dish[] {
  if (!region) return dishes
  
  const regionKey = region.trim()
  const allowedRegions = REGION_MAP[regionKey]
  
  // If region not in map OR Mix/General, 
  // return all dishes
  if (!allowedRegions || allowedRegions.length === 0) {
    return dishes
  }
  
  const filtered = dishes.filter(dish => {
    if (!dish.region || dish.region.length === 0) {
      return true // include dishes with no region tag
    }
    return dish.region.some(r => 
      allowedRegions.some(allowed => 
        r.toLowerCase().includes(allowed.toLowerCase()) ||
        allowed.toLowerCase().includes(r.toLowerCase())
      )
    )
  })
  
  // Safety fallback: if filter is too aggressive 
  // and returns less than 15 dishes, fall back to 
  // all dishes to avoid empty plans
  if (filtered.length < 15) {
    console.warn(
      `Region filter for "${region}" returned only 
      ${filtered.length} dishes. Falling back to 
      full dish pool.`
    )
    return dishes
  }
  
  return filtered
}
